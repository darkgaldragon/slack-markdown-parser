# セキュリティポリシー

## サポート対象バージョン

- `2.x`: サポート対象
- `1.x`: セキュリティ修正はベストエフォート

## 脆弱性報告

脆弱性は GitHub Security Advisories から非公開で報告してください。

- 推奨: `Security` タブ -> `Report a vulnerability`
- 使えない場合: メンテナーに非公開チャネルで連絡

報告時に含めてほしい情報:
- 影響バージョン
- 再現手順
- 影響範囲/深刻度
- 可能なら回避策・修正案

## 初動目標

- 初回応答: 72時間以内
- トリアージ/進捗連絡: 7日以内
- 修正時期: 深刻度と悪用可能性に応じて判断

## 秘密情報取り扱い

- APIキー/トークン/秘密鍵をコミットしない。
- リリース前と CI で秘密情報スキャンを実行する。
- 漏えい検知時は以下を必須とする:
  1. 公開/リリースを停止
  2. 資格情報を即時失効・ローテーション
  3. 必要に応じて履歴から削除
  4. インシデント内容と対策を記録
